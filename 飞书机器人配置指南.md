## 🔧 **配置步骤**

### **步骤 1：获取飞书 App ID 和 App Secret**

如果您还没有飞书应用，请按以下步骤创建：

1. **访问飞书开放平台**: https://open.feishu.cn/
2. **创建企业自建应用**
3. **获取应用凭证**:
   - **App ID**: 在"凭证与基础信息"中查看
   - **App Secret**: 在"凭证与基础信息"中获取

### **步骤 2：在系统中配置飞书信息**

您可以通过以下方式配置飞书信息：

#### **方式 A：通过环境变量配置 (推荐)**
编辑服务器配置文件 `.env`:

```bash
# 编辑 .env 文件
FEISHU_APP_ID=您的AppID
FEISHU_APP_SECRET=您的AppSecret
```

然后重启服务：
```bash
pkill -f "node dist/index.js"
cd server && PORT=3002 node dist/index.js
```

#### **方式 B：通过数据库设置 API**
使用以下 curl 命令配置：

```bash
# 获取认证token
TOKEN=$(curl -s -X POST https://rc.mboker.cn/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"admin@mboker.cn","password":"463136891"}' | jq -r '.accessToken')

# 配置飞书 App ID 和 App Secret
curl -X POST https://rc.mboker.cn/api/settings \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $TOKEN" \
  -d '{
    "feishu_app_id": "您的AppID",
    "feishu_app_secret": "您的AppSecret"
  }'
```

### **步骤 3：飞书开发者后台配置**

#### **1. 配置权限**
在飞书开发者后台，为应用添加以下权限：
- **contact:user:read**: 读取用户信息
- **im:message**: 发送和接收消息
- **task:task**: 创建任务提醒

#### **2. 配置事件订阅**
在"事件与回调"中：
- **启用事件订阅**
- **订阅方式**: 选择"使用长连接接收事件/回调"
- **订阅事件**:
  - `im.message.receive_v1` (接收消息)

#### **3. 配置消息与卡片**
在"消息与卡片"中：
- **启用消息接收**
- **配置机器人描述和头像**

### **步骤 4：绑定用户 OpenID**

飞书机器人需要知道哪个飞书用户对应系统用户。您需要：

1. **获取用户的飞书 OpenID**
2. **在系统中绑定**

可以通过以下 API 绑定：

```bash
# 绑定当前用户到飞书 OpenID
curl -X POST https://rc.mboker.cn/api/settings \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $TOKEN" \
  -d '{
    "feishu_openid_5f54a9c8-5a90-41bc-adda-9cf5c06031b7": "用户的飞书OpenID"
  }'
```

### **步骤 5：测试飞书机器人**

配置完成后，测试功能：

1. **发送消息测试**: 在飞书中@机器人发送消息
2. **创建任务测试**: 在系统中创建带提醒的任务
3. **接收提醒测试**: 测试任务到期提醒

## 🐛 **故障排除**

### **问题 1：机器人无响应**
- **检查**: 飞书服务是否启动 (查看 `server.log`)
- **解决**: 确认 App ID 和 App Secret 正确

### **问题 2：收不到消息**
- **检查**: 飞书开发者后台事件订阅配置
- **解决**: 确保启用了 `im.message.receive_v1` 事件

### **问题 3：无法发送消息**
- **检查**: 应用权限配置
- **解决**: 添加 `im:message` 权限

### **问题 4：用户绑定失败**
- **检查**: OpenID 是否正确
- **解决**: 使用正确的飞书用户 OpenID
